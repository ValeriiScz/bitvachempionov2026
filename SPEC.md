## 2. Структура данных турнира

Турнир состоит из:
- Квалификационных игр (обозначаются 1-N)
- Финала (обозначается 2-N)

В каждой игре несколько столов (Table 1, 2, 3). Каждый стол:
- 10 игроков
- У каждого: место (1-10), роль, результат, доп баллы
- Цвет шапки стола = цвет победителей

ВАЖНО про турнир 826: итоговая таблица не считается автоматически — судья закрывает турнир вручную. По ходу турнира доступны только результаты конкретных игр. Сводную таблицу нужно собирать самим из данных игр.

Первоубиенный — первый игрок покинувший стол (не обязательно с местом №1). На mafgame.org визуально выделен цветом в таблице.

ВЫВОД: турнирная сводка = агрегация игр на лету, не отдельная сущность.

## 3. Цветовая система

Команды:
- Красные (Red): Citizen + Sheriff. Цвет шапки выигравшего стола — красный.
- Чёрные (Black): Mafia + Don. Цвет шапки выигравшего стола — чёрный.

На сайте использовать те же цвета на индикаторе результата стола.

## 4. Колонки и термины

Колонка | Расшифровка
--------|------------
Σ (Total) | Итоговая сумма баллов = WPTS + APTS + CI (+ BM)
WPTS | Баллы за победу команды
APTS (Σap) | Сумма доп баллов
+ / - | Положительные / отрицательные доп баллы отдельно (для визуализации)
BM | Баллы за угадывание мафии из-под расстрела (только мирные, убитые первой ночью)
CI | Прогрессивный бонус за смерть первой ночью (защита от фарминга одного игрока)
W | Кол-во побед игрока в турнире
D | Кол-во побед за дона
S | Кол-во побед за шерифа
K | Сколько раз убит первой ночью

Минусы (отрицательные доп баллы):
- За мафию = чаще всего промах при стрельбе или плохая игра
- За мирного = разные причины: штраф за нарушение, плохая стрельба, etc

Английские термины (WPTS, APTS, CI, BM) — оставляем как есть, это стандарт спортивной мафии.

## 5. Градация доп баллов (APTS) — визуальная

Значение | Отображение
---------|------------
0 | Тёмно-серый, мелкий, не бросается в глаза
Минус | Красноватый, не яркий
+0.1 до +0.2 | Обычный текст
+0.3 до +0.4 | Жирный
+0.5 и выше | Яркая рамка или эмодзи-индикатор — это круто

## 6. Иконки ролей

Роль | Иконка | Примечание
-----|--------|------------
Citizen | сердечко | Мирный
Sheriff | шерифская звезда | Не путать с шляпой
Don | дон-шляпа (мафиозная) | Именно дон-шляпа, не любая шляпа
Mafia | пистолет | Чёрный member команды

Иконки минималистичные, рядом с аватаркой игрока. Стиль монохромный по умолчанию, цветной при необходимости.

Конкретный набор иконок — выбрать на этапе разработки. Варианты: emoji, Lucide icons, кастомный SVG.

## 7. Страницы сайта

Приоритет внедрения (от моего ВЫВОДА — нужно подтверждение):
1. /standings — самая ценная, её хотят все игроки
2. /player — второй приоритет
3. /analytics — потом
4. /results — потенциально объединить с /seating

### 7.1 /player — Профиль игрока
Клик по игроку → таблица истории:
- Колонки: Игра / Стол / Роль (иконка) / W или L / WPTS / APTS / CI
- Внизу: Total (сумма очков), текущее место в турнире
- Мини-подсказка: отрыв от игрока впереди, отрыв от игрока сзади

Референс — скриншот @mafgamer_bot (см. раздел 8).

### 7.2 /results — Рассадка с результатами (новая страница)
Текущий /seating не трогаем.
- Те же столы с аватарками
- По центру стола: индикатор результата (красный фон = город победил, чёрный = мафия)
- Рядом с аватаркой: иконка роли
- Под аватаркой: доп баллы с градацией из раздела 5

ВЫВОД: возможно объединить с /seating через переключатель "до игры / результаты". Спорно — обсудить.

### 7.3 /standings — Сводная таблица
- Колонки: Место / Игрок / Очки / Игр сыграно
- Игр сыграно ВАЖНО — не все играют все игры
- Сортировка по убыванию Очков
- Стиль: «как Telegram бот, но красиво» — моноширинный для цифр, чистая типографика

Референс — скриншот @mafgamer_bot (см. раздел 8).

### 7.4 /analytics — Аналитика
- Портрет среднего игрока: средний балл / победы / доп баллы
- Фильтр: все / только красные / только чёрные
- Топ-10 по доп баллам (все игры)
- Топ-10 лучших мирных (формула в открытых вопросах)
- Топ-10 лучших чёрных (формула в открытых вопросах)

## 9. Формулы (выведено из референсов)

FACT (из YeS Cyprus Open):
- Total Points = WPTS + APTS + CI (12.45 = 8.40 + 3.89 + 0.16)
- BM в карточке не показан — либо колонка скрывается когда везде 0, либо BM редко начисляется

FACT — дискретные значения WPTS:
- Стадия 1 (квалификация), победа: 0.75
- Стадия 2 (финал), победа: 0.98
- Поражение: обычно 0, но есть исключения (1-7 Red L = 0.75 — открытый вопрос)

ВЫВОД — стадии:
- 1-N = квалификация (коэффициент 0.75 для W)
- 2-N = финал (коэффициент 0.98 для W)
- Финал ценнее, но не сильно — это не плей-офф

ВЫВОД — CI начисляется не только мирным:
- У YeS CI=0.16 в игре 1-9 где он был Sheriff и проиграл
- ГИПОТЕЗА: CI = смерть первой ночью, бонус прогрессивный (увеличивается с каждым следующим разом)

## 11. Технический контекст

- Сайт: https://bitvachempionov2026.netlify.app
- Репо: https://github.com/ValeriiScz/bitvachempionov2026
- Текущий деплой: GitHub → Netlify автодеплой (настроено 23.05.2026)
- Стек: статический HTML/CSS/JS, без билда
- Данные: data.json в корне репо (планируется)

Существующие страницы:
- / — главная (выбор раздела)
- /seating.html — рассадка турнира 826 без результатов
- /cards.html — карточки 30 игроков 826 (статика по местам/слотам/совместным играм, без турнирных результатов)

Планируемые страницы (см. раздел 7):
- /standings.html
- /player.html
- /results.html
- /analytics.html

Парсинг mafgame.org:
- ФАКТ: сайт SPA, рендерится через JS. curl/wget возвращают пустую оболочку.
- РЕШЕНИЕ временное: Claude в Cowork через Computer Use (Chrome MCP) парсит, кладёт в data.json, пушит.
- ВЫВОД: для production нужен либо локальный Python+Playwright скрипт, либо GitHub Actions с headless Chromium — иначе зависим от того, что коворк-сессия активна.
